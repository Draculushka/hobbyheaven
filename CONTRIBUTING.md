# Руководство по внесению изменений

Это руководство содержит стандарты и процессы, принятые в проекте HobbyHold. Соблюдение этих правил помогает поддерживать чистоту кода и упрощает поддержку проекта.

---

## Требования к среде разработки

1.  **Python:** Версия 3.12 или выше.
2.  **Зависимости:** Установка производится строго через `pip install -r requirements.txt`.
3.  **Docker:** Наличие Docker и Docker Compose для тестирования инфраструктуры.

---

## Процесс разработки

1.  **Создание ветки:** Все изменения должны вноситься в отдельные ветки.
    *   `feature/название-функции` — для новых возможностей.
    *   `bugfix/название-бага` — для исправления ошибок.
    *   `docs/описание` — для правок документации.
2.  **Линтинг:** Рекомендуется придерживаться стандарта PEP 8. Перед коммитом проверьте код на отсутствие неиспользуемых импортов и явных синтаксических ошибок.
3.  **Проверка:** Если вы добавляете новый эндпоинт, убедитесь, что он отображается в Swagger UI (`/docs`) и возвращает корректные статус-коды.

---

## Стандарты коммитов

Мы используем упрощенный формат Conventional Commits для описания изменений:

*   `feat:` — добавление нового функционала.
*   `fix:` — исправление ошибки.
*   `docs:` — изменения в документации.
*   `refactor:` — правки кода без изменения логики (улучшение читаемости).
*   `chore:` — обновление зависимостей или конфигурационных файлов.

**Пример:** `feat: добавить поддержку загрузки нескольких изображений`

## Работа с базой данных (Миграции)

Мы используем **Alembic** для управления структурой базы данных. Это позволяет изменять модели без потери данных.

### Если вы изменили `app/models.py`:
1.  **Создайте миграцию:**
    ```bash
    export PYTHONPATH=$PYTHONPATH:.
    alembic revision --autogenerate -m "описание изменений"
    ```
    Это создаст файл в `migrations/versions/`. Проверьте его содержимое.

2.  **Примените изменения к базе:**
    ```bash
    alembic upgrade head
    ```

### Полезные команды:
*   `alembic current` — узнать текущую версию базы.
*   `alembic history` — посмотреть список всех миграций.
*   `alembic downgrade -1` — откатить последнее изменение.
