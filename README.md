# HobbyHold

HobbyHold — это платформа для публикации и обмена хобби. Проект представляет собой полноценное веб-приложение, построенное на современном стеке технологий с использованием многослойной архитектуры (Layered Architecture).

---

## Основные возможности

*   **Web UI:** Современный интерфейс на Tailwind CSS для просмотра, создания и поиска хобби.
*   **Многослойная архитектура:** Четкое разделение ответственности между API, бизнес-логикой и настройками.
*   **Поиск по синонимам:** Умный поиск, понимающий разные названия одного и того же увлечения.
*   **Безопасность:** JWT-авторизация (HttpOnly Cookies), хеширование паролей и очистка HTML (bleach).
*   **Загрузка изображений:** Поддержка прикрепления фотографий к постам с автоматической очисткой старых файлов.
*   **Docker Ready:** Запуск всей инфраструктуры одной командой через Docker Compose.

---

## Стек технологий

*   **Backend:** FastAPI (Python 3.12)
*   **Database:** PostgreSQL + SQLAlchemy (ORM)
*   **Templates:** Jinja2 + Tailwind CSS
*   **Testing:** Pytest + HTTPX
*   **Infrastructure:** Docker, Docker Compose, Nginx
*   **Migrations:** Alembic

---

## Архитектура проекта (Layered Architecture)

Проект переведен на модульную структуру для обеспечения масштабируемости и тестируемости:

*   **`app/api/`**: Слой обработки HTTP-запросов (маршруты).
*   **`app/services/`**: Слой бизнес-логики (сохранение данных, обработка изображений, теги).
*   **`app/core/`**: Глобальные настройки, конфигурация безопасности и JWT.
*   **`app/models.py`**: Описание структуры таблиц БД.
*   **`tests/`**: Набор автоматизированных тестов (Unit и Integration).

```text
hobbyhold/
├── app/
│   ├── api/             # Эндпоинты (auth, hobbies)
│   ├── services/        # Бизнес-логика
│   ├── core/            # Настройки и безопасность
│   ├── main.py          # Точка входа (сборка приложения)
│   ├── models.py        # Модели БД
│   ├── database.py      # Подключение к БД
│   ├── templates/       # HTML шаблоны
│   └── uploads/         # Загруженные изображения
├── tests/               # Автоматизированные тесты (pytest)
├── nginx/               # Конфигурация Nginx
└── migrations/          # Миграции базы данных (Alembic)
```

## Быстрый старт (Docker)

1.  **Запустите сервисы:**
    ```bash
    docker-compose up --build
    ```
2.  **Приложение будет доступно по адресу:** [http://localhost](http://localhost)

---

## Тестирование

Для запуска тестов используйте `pytest`. Тесты автоматически используют изолированную базу данных SQLite в памяти.

```bash
# Активируйте venv и установите зависимости
pip install -r requirements.txt
export PYTHONPATH=$PYTHONPATH:.
pytest tests/
```

---

## Локальная разработка (без Docker)

1.  **Настройте виртуальное окружение и установите зависимости:**
    ```bash
    python -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
    ```
2.  **Запустите сервер:**
    ```bash
    uvicorn app.main:app --reload
    ```

## Безопасность и Стандарты разработки

1.  **Разделение ответственности:** Логика обработки данных должна находиться в `services`, а не в роутах.
2.  **Миграции:** Любое изменение БД выполняется только через `alembic revision --autogenerate`.
3.  **Тесты перед коммитом:** Всегда запускайте `pytest`, чтобы убедиться, что изменения не сломали существующий функционал.
4.  **Stateless API:** Мы стремимся к тому, чтобы сервер не хранил состояние сессий в памяти, используя JWT в HttpOnly куках.
